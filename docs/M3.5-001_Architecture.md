## 2. 架构说明

### 2.1 与既有关卡的边界（核心设计判断）

M3-001 的 `KnowledgeGraph` 是 **per-topic** 图：它把每个关系端点解析为 **local id**，并把跨主题（`namespace:id`）边当作 "dangling-by-design" **丢弃**（`graph.py:161-166`）。这导致 M3-003 虽然数据已互联，但"点击罗马 → 直接看到埃及/丝路邻居"在**图层面**做不到。

`GlobalGraph` 是 `KnowledgeGraph` 的**互补层**，不是替代品：
- 它把每个端点通过 `KnowledgeRegistry.resolve()` 解析为**规范 `global_id`**；
- 因此跨主题边被**纳入**而非丢弃；
- 结果是一张覆盖全互联世界的**统一邻接图**。

### 2.2 GlobalGraph 设计（`core/global_graph.py`）

- **全局节点注册**：遍历全部 topic 的每个 entity，以 `global_id` 为规范键注册为 `GlobalNode`（topic / local_id / type / name / entity）。无关系的实体也注册。
- **跨主题边纳入**：遍历全部 relationship，端点经 `registry.resolve()` 解析为 `EntityRef`，再取规范 `global_id` 作为边端点；跨主题边自然连入。悬空端点（解析失败）跳过——属 validation 关注，非图关注。
- **复用而非重造**：邻接 / BFS / 最短路径直接委托既有 `DirectedGraph`（`graph.py`），`GlobalGraph` 只是"跨主题感知"的薄层。
- **节点键**：`global_id`；实体缺 `global_id` 时回退 `topic:local_id`（M3-003 互联数据所有实体均有 `global_id`，不会触发回退）。
- **对外能力**：`neighbors(global_id, direction)`（out/in/both）、`find_path(src, tgt)`（最短路径，返回 global_id 列表）、`subgraph(roots, max_depth)`（返回受限的新 `GlobalGraph`，供 M3.5-002/003 圈定邻域）。

### 2.3 KnowledgeService 接线（additive，无新 API）

构造时一次性构建：`self._global_graph = GlobalGraph(self._topic_datasets, self._registry)`。
新增 4 个**库方法**（不新增 REST 端点，API 契约冻结）：

| 方法 | 作用 |
|---|---|
| `get_global_graph()` | 返回 `GlobalGraph` 实例 |
| `find_global_path(src_gid, tgt_gid)` | 跨主题最短路径 |
| `global_neighbors(global_id, direction)` | 原始（未排序）邻居，带对方节点信息 + 关系类型 + 方向 |
| `global_subgraph(roots, max_depth)` | 受限子图 |

> 排序 / 推荐 / UI / API 新增**均不在本 checkpoint**（属 M3.5-002/003/004）。`global_neighbors` 刻意返回**未排序**原始邻居，排序交给 M3.5-002。

## 5. 下一 checkpoint 建议：M3.5-002 Exploration Engine

**目标**：在 GlobalGraph 之上实现确定性 Exploration Engine。

- **输入**：`KnowledgeService.get_global_graph()` 已就绪。
- **范围**（来自冻结）：路径发现（BFS / 最短路径，复用 `DirectedGraph`）、四维评分排序（实体类型 / 时间邻近 / 关系方向 / 关系类型权重）、时间因素（用 `TimeValue.value`，BCE 为负）、关系权重兜底（null → 1.0）。**可解释性**：每条推荐附"为什么"（关系类型 + 路径）。
- **不实现**：AI 摘要、个性化推荐、GIS、写入。
- **建议拆分**：先 `core/exploration_engine.py`（纯算法，单测驱动），再由 M3.5-003 经 `KnowledgeService` + `/api/v1`（方案 A additive）暴露，最后 M3.5-004 前端原型。
- **并行机会**：M3-004 Search v2 可与 M3.5-002 并行（独立）；Data Agent 可同步扩写 `inherited`/`conquered`/`spread` 关系数据以喂养排序效果。
