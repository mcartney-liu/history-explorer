# Documentation Archive (90_ARCHIVE)

> **This directory contains historical documents. It is NOT an authoritative source.**
> Refer to `docs/INDEX.md` for current documentation.

This archive preserves early project documents that have been superseded by the
current documentation system (established in M8.8). Contents are retained for
historical traceability only and MUST NOT be treated as current architecture,
product, or engineering truth.

## Structure

- `legacy_structure/` — content migrated from the old `docs/` numbering system
  (`00_Project` … `11_Release`, `adr`, `api`, `repository_memory`).
- `root_design_docs/` — early root-level design explorations (M1–M3 era) that were
  replaced by the current L1–L6 documentation architecture.

## Archive Policy

- 90_ARCHIVE is **historical only**.
- The authoritative reading order is defined in `docs/INDEX.md`.
- Do not add new content here; new docs go into the active layers (00_VISION … 30_TEAM).
- Milestone reports (M3–M8.6) are NOT yet migrated here — migration is a pending
  follow-up task; they currently remain at `docs/` root.
