# History Explorer - Information Architecture

Version: 1.0

Status: Active


# 1. Overview

History Explorer organizes historical knowledge as a connected exploration system.

The core idea:

History is not a collection of isolated facts.

History is a network of:

- Events.
- People.
- Civilizations.
- Locations.
- Time periods.
- Relationships.


The information architecture supports:

Explore → Connect → Understand → Discover


# 2. Core Knowledge Entities


# 2.1 Event

Definition:

A historical occurrence or process.

Examples:

- Wars.
- Political changes.
- Scientific discoveries.
- Cultural movements.
- Economic transformations.


Key attributes:

- Name.
- Time period.
- Location.
- Participants.
- Causes.
- Consequences.


# 2.2 Person

Definition:

A historical individual connected with events or civilizations.


Key attributes:

- Name.
- Lifetime.
- Roles.
- Related events.
- Related civilizations.


# 2.3 Civilization

Definition:

A human society or cultural system.

Examples:

- Ancient civilizations.
- Empires.
- Nations.
- Cultural groups.


Key attributes:

- Time period.
- Geographic area.
- Culture.
- Political structure.
- Related events.


# 2.4 Location

Definition:

A geographic place related to historical activities.


Key attributes:

- Name.
- Geographic coordinates.
- Region.
- Related events.
- Related civilizations.


# 2.5 Time Period

Definition:

A historical time structure.

Examples:

- Ancient era.
- Medieval period.
- Modern period.
- Specific historical periods.


Key attributes:

- Start time.
- End time.
- Historical characteristics.


# 3. Relationship Model


History Explorer focuses on meaningful connections.


Core relationships include:


## Event Relationships

Examples:

- Event caused Event.
- Event influenced Event.
- Event happened before Event.
- Event happened during Time Period.


## Person Relationships

Examples:

- Person participated in Event.
- Person influenced Civilization.
- Person related to Person.


## Civilization Relationships

Examples:

- Civilization interacted with Civilization.
- Civilization controlled Location.
- Civilization influenced Culture.


## Location Relationships

Examples:

- Event happened at Location.
- Civilization existed in Location.


# 4. Exploration Dimensions


All exploration follows four dimensions:


## Time

Understanding:

When did it happen?


## Space

Understanding:

Where did it happen?


## Entity

Understanding:

Who or what was involved?


## Relationship

Understanding:

How are things connected?


# 5. Knowledge Exploration Structure


Users can explore through:


## Event First

Example:

A user starts from a historical event.

↓

Discover people.

↓

Discover civilizations.

↓

Discover related events.


## Person First

Example:

A user starts from a historical person.

↓

Discover achievements.

↓

Discover historical context.


## Civilization First

Example:

A user starts from a civilization.

↓

Explore timeline.

↓

Explore conflicts and interactions.


## Location First

Example:

A user starts from a location.

↓

Discover historical events.

↓

Discover civilizations.


# 6. Information Principles


The architecture follows:


1. Connected knowledge over isolated articles.

2. Context over simple facts.

3. Relationships over lists.

4. Exploration paths over fixed reading order.

5. Historical complexity over oversimplification.
