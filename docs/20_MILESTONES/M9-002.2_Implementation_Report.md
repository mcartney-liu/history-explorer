## 3. API Integration

- **Endpoint consumed** (already shipped in v0.11.0): `GET /entity/{id}/recommendations?limit=5&seen=<csv>`
- **Base URL**: inlined `import.meta.env.VITE_API_BASE || 'http://localhost:8000'` (byte-identical to `App.tsx` L54) — zero new API-client module, zero new dependency.
- **`fetchRecommendations(entityId, seenGlobalIds: Set<string>, limit)`** pure helper:
  - Builds URL with `encodeURIComponent` on both path segment and the `seen` CSV.
  - Maps responses: `200` → parsed `RecommendationResult`; `404` → `ErrorKind 'notfound'`; `fetch` throw → `'network'`; other non-ok / unparseable → `'parse'`.
- **Data flow**: `App` passes `seenGlobalIds` (derived from `recent`, `App.tsx` L417–419) → helper sends as `seen` param → engine returns ranked `recommendations[]` → view renders in engine order (no client re-rank).
- **Navigation**: card click → `onNodeClick(global_id)` → wired to existing `openEntity` in `App.tsx` → `navigateTo` → `fetchNode` → `GET /entity/{id}`. **Exploration loop closes with zero new navigation code.**

## 4. UX Flow

Position in the entity view (Candidate C, per M9-002 Design):
1. **What it is** — `EntityPage`
2. **Where to go next, and why** — `RecommendationPanel` (NEW): titled "下一站探索"; each card shows `target_entity.name` + `type` badge, the engine's `reasons[]` ("why"), and the `relation_path[]` (`from → relationship → to`, with direction/weight in a tooltip).
3. **Where else you can go** — `ContinueExploringPanel` (neutral engine-ranked fallback, unchanged).

States:
- **Loading** → `LoadingSkeleton` ("推荐探索路径…").
- **Error** → `ErrorCard` (kind-aware copy) with a **Try again** retry that re-invokes the fetch.
- **Empty** (`recommendations.length === 0`) → renders `null` (consistent with `ContinueExploringPanel`).
- **Seen** → already-visited nodes get an `is-seen` marker; order is **never** changed.

## 5. Tests

**Harness**: `vitest ^4.1.10`, `environment: 'node'`, `renderToStaticMarkup` (no jsdom, no `@testing-library`, no new deps). 9 new tests in `RecommendationPanel.test.tsx`:

1. `fetchRecommendations` builds the correct URL (`/entity/.../recommendations?limit=5&seen=`) and parses the result; `seen` global ids survive URL-encoding.
2. 404 → `notfound` error kind.
3. Network failure (fetch throws) → `network` error kind.
4. Non-ok / unparseable response → `parse` error kind.
5. `RecommendationPanelView` renders title ("下一站探索"), entity name, type, and `reasons[]`.
6. Engine order preserved (no client re-rank).
7. Already-seen nodes marked without reordering.
8. Route-compatibility contract: each card's `aria-label` encodes the engine target (wired to `onNodeClick`→`openEntity`).
9. Container renders a loading skeleton in static markup without invoking fetch (useEffect not run server-side).
