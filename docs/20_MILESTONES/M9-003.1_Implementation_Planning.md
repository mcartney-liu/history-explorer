## 1. Goal

在纯前端、零冻结触碰的前提下，把 M9-001/002 已经具备的"确定性下一站推荐 + 可解释理由"沉淀为一条**可回溯、带"为何抵达"注解**的探索旅程视图（Exploration Journey）。交付：
- 新组件 `ExplorationJourney.tsx`（纯消费者，三层结构）。
- App 新增 session 级注解 Map `journeyReasons`（仅推荐点击时填充，永不入 `navigation.ts`）。
- `RecommendationPanel.onNodeClick` 向后兼容扩展为 2 参（携带推荐上下文）。

## 2. Prerequisites (verified at planning time)

- `master` @ v0.12.0（`f105089`，M9-002 doc archive）为基础。
- `RecommendationPanel` 已发布（v0.12.0），其 `onNodeClick(gid)` 为 1 参；本计划将其扩展为 `(gid, ctx?)` 且 1 参调用方零改动。
- `navigation.ts` 的 `history` / `cursor` / `goTo` 为唯一导航真相源，本计划**只读**消费。
- `frontend/package.json` 版本仍为 `0.10.0`（已知不一致，归 M9-006，本里程碑**不 bump**）。

## 4. Insertion Points (校准)

- **App.tsx entity 分支**：在现有 `ExplorationTrail` 之后、`RecommendationPanel` 之前，插入 `<ExplorationJourney history={history} cursor={cursor} journeyReasons={journeyReasons} onStepClick={goTo} />`。
- **App.tsx `goHome`**：追加 `setJourneyReasons(new Map())`（session 边界）。
- **RecommendationPanel 卡片点击**：`onClick={() => onNodeClick?.(gid, buildRecommendationContext(rec))}` —— 1 参调用方行为不变（`ctx` 为可选）。
- **App `onNodeClick` 接收端**：`(gid, ctx) => { if (ctx) setJourneyReasons(...); openEntity(gid, ...) }` —— 直接导航（无 ctx）不注解。

## 5. Key Invariants (must hold after implementation)

1. `ExplorationJourney` **owns no navigation state** — no `pushHistory`, no `goTo` inside the component.
2. `journeyReasons` **is an annotation map, not a navigation stack** — never enters `navigation.ts`.
3. `ExplorationTrail` **remains unchanged** — Journey is a parallel sibling.
4. `RecommendationPanel` 1-arg `onNodeClick` callers **keep working** (backward compatible).
5. Empty journey (`history.length < 2`) → component returns `null`.
