# History Explorer Development Guide


## Development Philosophy

Documentation before implementation.

Small iterations before large features.

Maintainable architecture over quick solutions.


## Project Structure

frontend:
Frontend application.

backend:
Backend services.

ai:
AI capabilities.

data:
Historical data systems.

scripts:
Development tools.


## Development Rules

1. Follow project architecture.

2. Keep changes focused.

3. Avoid unnecessary complexity.

4. Commit changes clearly.

5. Update documentation when architecture changes.


## Running Development Environment

Frontend:

```bash
npm install
npm run dev
```

Backend:

```bash
pip install -r requirements.txt
uvicorn app.main:app --reload
```
